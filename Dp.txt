A Markov Decision Process (MDP) can be solved using Dynamic Programming (DP) via algorithms like Value Iteration,
which recursively updates state values until they converge to the optimal policy.Below is a complete,
 step-by-step math evaluation and a runnable Python implementation
 for a sample 3-State Gridworld MDP.1. Define the MDP EnvironmentConsider a linear world with 3 states: \(S = \{s_1, s_2, s_3\}\).s₁: Starting State.s₂: Middle State.s₃: Terminal State (Goal)
 .Available Actions (A)From any non-terminal state, the agent can choose to move Right or Left.Transition Probabilities (P)The environment is stochastic (slippery).
  The intended direction succeeds with 80% probability; it slips in the opposite direction with 20% probability.\(P(s_2 \mid s_1, \text{Right}) = 0.8\),
   \(P(s_1 \mid s_1, \text{Right}) = 0.2\) (bounces off the wall).\(P(s_3 \mid s_2, \text{Right}) = 0.8\), \(P(s_1 \mid s_2, \text{Right}) = 0.2\).
   Rewards (R)Reaching the terminal state s₃ yields a reward of +10.All other steps yield a penalty of -1 to encourage the fastest path.HyperparametersDiscount factor
    γ = 0.92. Value Iteration Math FormulationValue Iteration applies the Bellman Optimality Equation as an iterative update:\(V_{k+1}(s)=\max _{a\in A}
    \sum _{s^{\prime }}P(s^{\prime }\mid s,a)\left[R(s,a,s^{\prime })+\gamma V_{k}(s^{\prime })\right]\)Let's initialize all states at V₀(s) = 0.
    Because s₃ is a terminal state, its value remains fixed at V(s₃) = 0; its reward is collected on the incoming transition.Iteration 1 (k=1)
    For State s₁:Action Right: 0.8 ⋅ (-1 + 0.9 ⋅ 0) + 0.2 ⋅ (-1 + 0.9 ⋅ 0) = -1.0Action Left: 1.0 ⋅ (-1 + 0.9 ⋅ 0) = -1.0
     (hits left wall completely)\(V_1(s_1) = \max(-1.0, -1.0) = -1.0\)For State s₂:Action Right: 0.8 ⋅ (+10 + 0.9 ⋅ 0) +
      0.2 ⋅ (-1 + 0.9 ⋅ 0) = 8.0 - 0.2 = 7.8Action Left: 0.8 ⋅ (-1 + 0.9 ⋅ 0) + 0.2 ⋅ (+10 + 0.9 ⋅ 0) = -0.8 + 2.0 = 1.2\(V_1(s_2) =
      \max(7.8, 1.2) = 7.8\)Iteration 2 (k=2)For State s₁:Action Right: 0.8 ⋅ (-1 + 0.9 ⋅ V₁(s₂)) + 0.2 ⋅ (-1 + 0.9 ⋅ V₁(s₁))
      Action Right = 0.8 ⋅ (-1 + 0.9 ⋅ 7.8) + 0.2 ⋅ (-1 + 0.9 ⋅ (-1.0)) = 0.8 ⋅ (6.02) + 0.2 ⋅ (-1.9) = 4.816 - 0.38 = 4.436V₂(s₁) = 4.436
